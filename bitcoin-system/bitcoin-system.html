Blockchain

Blockchain is a network of people that cooperate together to update the ledger.

  • The ledger is held by every peer which we called the “Node” in the Bitcoin network.
    • A ledger is a record of all transactions
  • When a transaction happens, every node in the network will update its ledger and that’s how the system tracks your balance.

Bitcoin VS. Reality

There is a differences of how Bitcoin handling the ledger compare to the real life.

  • How does ledger handle in real life?
    • The ledger is always held by a trustworthy authority like a bank (Trust required).
  • How does Bitcoin handle the ledger?
    • The system is based on mathematical functions and protocols where no trust is required.
  • How is it significant?
    • When a bank declares bankruptcy or problems happen to the bank, it is challenging to get your money back. Compared to the decentralized system like Bitcoin, everyone holds a ledger so your account is always somewhere in the system.

HASH signature

HASH signature is a digital signature system that will always produce a fixed output

  • a cryptographic function that turns any lengths of input into a fixed size of the output.
    • Given output can’t backsolve for input (one-wayed function).
    • Every input should have a unique output

    • A great website to try it out SHA-256 generator

  • HASH collision
    • The output has a definite length, there might be inputs that have the same output

Identity verification

How does Bitcoin vertify a client's identity?

When registering for a Bitcoin wallet, it generates a unique but connected pair of keys, the public key, and the private key. The private key should be secret and nobody should know except the owner, but the public key is public for transactions. The private key is connected with the public key by SHA-256, which is one-way and people can't go back-solve it.

  • Private key: encrypting the message
  • Public key: decrypting the message
